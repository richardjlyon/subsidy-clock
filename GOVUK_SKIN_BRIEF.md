# Design brief — "GOV.UK" skin for The Subsidy Clock

**For:** Claude Design
**From:** The Subsidy Clock team
**Working title:** `govuk-skin` (a selectable skin, same pattern as `energytrap-skin`)

---

## 1. The one-line brief

Dress the Subsidy Clock in the full livery of an official UK government
service — black crown masthead, GDS Transport type, blue underlined links,
green "Start now" buttons, the lot — as if **the Department for Energy
Security & Net Zero had finally published the subsidy total themselves.**

The joke is deadpan, not loud: *they won't do it, so we did — in their own
house style.* The humour comes entirely from the contrast between austere,
form-filling officialdom and a £226,966,081,251 number ticking up in the
corner. We never break character in the chrome; we only wink in the copy.

## 2. Tone & guardrails (read first)

- **Deadpan civil-service register.** Plain English, short sentences, passive
  where a real notice would use it. "This service tells you how much…", "Last
  updated", "Report a problem with this page". Let the number do the shouting.
- **Gently mocking, never sneering.** The target is the *absence* of an
  official tracker, not individuals. One dry wink is funnier than five.
- **This is satire and must not pass as a real government service.** That is
  both an ethics line and a legal one:
  - The **Royal Arms / GOV.UK crown logo, the "GOV.UK" wordmark, and the GDS
    Transport typeface are Crown property** and restricted to genuine
    government services. Do **not** ship the exact protected assets.
  - Use a **recognisable pastiche**: a generic crown-ish glyph (not the Royal
    Arms), and a wordmark that reads as a near-miss — e.g. `GOV.UK` set in the
    block style but paired with the service name so it's clearly unofficial,
    or a tiny "unofficial" tag. A near-miss is both safer and funnier.
  - The footer **must** carry a plain disclaimer (see §8). Keep the *visual*
    grammar of Crown copyright / Open Government Licence, but the words must
    say this is **not** an official publication.
  - For the font, prefer the official fallback stack (`Arial`) or an open
    Transport clone ("Transport New"), **not** licensed GDS Transport.

## 3. How to build it (architecture)

This is a **CSS-led reskin**, exactly like `energytrap-skin`:

- The whole site is driven by CSS custom properties + component classes in
  `site/style.css`. ~90% of this skin is **re-pointing those tokens** to
  GOV.UK values (§4) and restyling a handful of components (§6).
- A small amount of **markup furniture** must be added to each page's
  `<header>`/`<footer>` (masthead, phase banner, breadcrumbs, OGL footer,
  feedback bar) — §5. These are static HTML edits to `index.html`,
  `methodology.html`, `data.html`, `about.html`, `corrections.html`, and the
  explainer pages.
- **Explainer pages load their own `site/explainers/explainer.css`** — it
  carries a separate copy of the palette/fonts and must be updated too (this
  bit the last reskin). Treat it as a second token block.
- **Leave the embed widget (`site/embed/widget.html`) in a neutral style** —
  it is generated by Python from a template and is embedded on third-party
  sites; the GOV.UK chrome would look broken out of context. (Same call the
  Energy Trap skin made.)
- Ship on a branch so it toggles via `git checkout`, like the existing skins.

## 4. Design tokens — map these onto the existing variables

The current `:root` in `site/style.css` defines the Energy Trap palette.
Re-point each token. Left = existing variable, right = GOV.UK value.

| `--paper-dim`           | `#f3f2f1`    | GOV.UK light-grey (section bands, table zebra is **not** used). |
| `--ink`, `--money`      | `#0b0c0c`    | Near-black text **and** figures. The number is black, not coloured. |
| `--muted`               | `#505a5f`    | GOV.UK secondary text grey (captions, "Last updated"). |
| `--faint`               | `#6f777b`    | |

| `--navy`, `--navy-deep` | `#0b0c0c`    | Masthead is **black** `#0b0c0c`, not navy. |

> The Design System's v6 refresh shifted some hexes (link `#1a65a6`, green
> `#0f7a52`, red `#ca3535`). **Use the classic values above** — recognisability
> is the entire point of the parody, and the classic blue/green/black is what
> the public pictures when they think "gov.uk".

### Chart series palette (replaces the amber→navy ramps)

GOV.UK has **no official chart component**, but statistical publications draw
from a fixed categorical web palette. Re-point the ramp tokens to it and
**drop the "headline = amber" idea** — in GOV.UK world nothing is highlighted,
everything is just a labelled category:

```
--c-ro:    #1d70b8   (blue)        --c-cm:    #28a197   (teal)
--c-fit:   #003078   (dark blue)   --c-ccl:   #6f72af   (light purple)
--c-cfdr:  #5694ca   (light blue)  --c-ets:   #f47738   (orange)
--c-cfdl:  #4c2c92   (purple)      --c-tnuos: #b58840   (brown)
--c-con:   #d4351c   (red)         --c-bsuos: #85994b   (light green)
--c-other: #b1b4b6   (grey — "all other costs")
```

GOV.UK guidance actively warns against too many colours — so an optional,
funnier variant: render **every series in one blue** (`#1d70b8`) and rely on
the legend + a "View this data as a table" toggle (§7). Maximum bureaucratic
restraint. Offer both; recommend the categorical set for legibility.

### Typography

- **Family:** `--sans: "GDS Transport", Arial, sans-serif;` but per §2 ship
  with `Arial`/"Transport New" — do **not** self-host licensed GDS Transport.
  **Drop Equity/Concourse and the serif entirely** — GOV.UK is sans-only.
  Set `--serif` to the same sans stack so reading text is also Arial.
- **Type scale** (desktop; line-heights are multiples of 5px for vertical rhythm):

  | Role            | size / line-height |
  |-----------------|--------------------|
  | xl heading      | 48 / 50, **bold** |
  | l heading       | 36 / 40, bold |
  | m heading       | 24 / 30, bold |
  | s heading       | 19 / 25, bold |
  | body            | 19 / 25 |
  | body-s / caption| 16 / 20 |

- Headings **bold (700)**, left-aligned. Body 19px, generous leading. No
  italics, no small-caps, no letter-spacing tricks. `font-feature-settings`
  keep `tnum` on for figures (GOV.UK uses lining tabular numerals).

### Layout & spacing

- **Pure white background, left-aligned, single column.** Kill all centering.
- `--radius: 0` — GOV.UK has **square corners** everywhere (cards, bars,
  buttons, inputs). This is a defining trait; do not round anything.
- Container max-width ~**960px** content (GOV.UK wraps prose to **two-thirds**,
  ~`660px`, for readability — apply this to `.card-intro`, `.prose`, captions).
- Spacing scale (use these step values): `5 / 10 / 15 / 20 / 30 / 40 / 50 / 60px`.
- Section dividers: a single `1px solid #b1b4b6` rule, not a card shadow.
  **Remove card drop-shadows and the warm card fill** — GOV.UK "cards" are
  just content blocks on white, separated by rules or whitespace.

## 5. Page furniture to add (the GOV.UK shell)

Wrap every page in the standard template. Add to the static HTML:

1. **Black masthead bar** (`#0b0c0c`, full-bleed, white text):
   `[crown glyph] GOV.UK` block top-left (pastiche per §2). Replaces the
   current `.masthead` brand block.
2. **Service navigation bar** below it (white, `#b1b4b6` bottom border):
   the service name — **"Energy subsidy spending tracker"** — as a bold link,
   with the nav links (Data / Methodology / About / Share) right-aligned.
3. **Phase banner** (a `BETA` tag — white text on `#1d70b8`, square — + a line
   of text in a `#b1b4b6`-bottom-bordered strip):
   > **BETA**  This is a new service. We built it because no one official
   > would. [Report a problem](#) to help us improve it.
4. **Breadcrumbs** under the banner, in GOV.UK style (chevron separators,
   small grey links):
   `Home › Environment › Renewable and low carbon energy › Subsidy spending tracker`
5. **Footer** (`#f3f2f1` background, `#1d70b8` 10px-ish top border is *not*
   standard — use the plain footer): two-column-ish support links row, then a
   **crown glyph + Open Government Licence boilerplate** *re-worded as a
   disclaimer* (see §8), with the OGL/crown set in the authentic visual style.
6. **"Is this page useful?" feedback bar** above the footer (a `#b1b4b6`
   top-bordered block): `Is this page useful?  [Yes] [No]  [Report a problem
   with this page]`. Pure set-dressing; the Yes/No can be inert or link to the
   existing corrections flow. This is one of the funniest, most-recognisable
   GOV.UK tells — include it.

## 7. Graphs & tabular data (the part to get right)

The user specifically wants this nailed. Two principles, both authentically GOV.UK:

**Tables are first-class; charts are secondary.** GOV.UK statistical
publications lead with accessible HTML tables and treat charts as an optional
visual. So:

- Style **`.data-table`** to the exact GOV.UK table spec:
  - White background, no outer border, **no vertical borders, no zebra striping.**
  - Each row separated by `border-bottom: 1px solid #b1b4b6`.
  - Header cells: **bold**, left-aligned (numeric headers right-aligned),
    same bottom border.
  - Cell padding `10px 20px 10px 0`; first column flush-left.
  - Numeric columns (`.num-col`) **right-aligned** with tabular lining numerals.
  - A bold **`<caption>`** above each table (e.g. "Largest recipients, by
    cumulative payment") — GOV.UK always captions tables.
- For **every chart, add a `<details>` "View this data as a table" toggle**
  beneath it (GOV.UK accessibility convention). The disclosure summary reads
  "Show this data as a table"; inside is a `.data-table` of the same numbers.
  This is both authentic *and* a great deadpan beat — the government way is to
  bury the drama in a collapsible table.

**Charts: flat, restrained, sourced.** There is no official GOV.UK chart
component (state this in the handoff so Design infers correctly), but the
consistent statistical-publication style is:

- No gradients, no 3D, no rounded bars, no drop-shadows, no animation.
- Categorical fills from the §4 palette (or all-one-blue for max restraint).
- Thin light-grey gridlines (`#b1b4b6`), a black zero line, black axis labels
  in the sans face, axis titles where helpful.
- A short, literal chart title as a bold heading **above** the chart, and a
  small grey **"Source: …"** caption **below** it. Always cite the source —
  that's the most GOV.UK habit of all.
- Plain text legend with square swatches; no interactivity beyond the toggle.

## 8. Copy rewrites (sample deadpan strings)

Keep these dry. A few examples to set the register — Design can extend:

- Service name: **"Energy subsidy spending tracker"**
- Hero caption: "Total cost of UK subsidies paid to low-carbon electricity
  generators since 2002."
- Phase banner: "**BETA** — This is a new service. We built it because no one
  official would. Help us improve it."
- "Last updated 16 June 2026. **Next update:** when someone official admits the figure."
- Methodology page intro: "Use this service to find out how much you and other
  bill-payers have paid to subsidise low-carbon electricity."
- Footer disclaimer (**required**, replaces real Crown copyright wording):
  > This is **not** an official government service and is not affiliated with
  > GOV.UK, the Crown, or any department. It is an independent, sourced record
  > presented in the style of one. All figures trace to official sources.
  Keep the OGL/crown *visual* furniture beside it, but the words make the
  status unmistakable.
- Feedback bar: "Is this page useful?  Yes / No" + "Report a problem with this page".
